# Feedback

> If you see unfamiliar placeholders or need to check which tools are connected, see [CONNECTORS.md](../CONNECTORS.md).

Provide constructive, specific feedback on student assignments, essays, projects, or presentations. Focus on growth-oriented comments that guide improvement.

## Usage

```
/feedback <student work or assignment description>
```

Examples:
- `/feedback [paste essay or assignment]`
- `/feedback student science lab report`
- `/feedback math problem set`

## Workflow

### 1. Analyze the Work

Read and understand:
- **Assignment requirements**: What was asked?
- **Student work**: What was submitted?
- **Context**: Grade level, learning goals, time constraints

### 2. Apply Feedback Framework

Use the **feedback-framework** skill to structure comments:

#### Start with Strengths
- Identify 2-3 specific things done well
- Connect to learning objectives
- Be specific, not generic

#### Address Growth Areas
- Focus on the work, not the student
- Be specific about what needs improvement
- Suggest actionable strategies

#### End with Encouragement
- Express confidence in student's ability
- Suggest next steps
- Invite questions

### 3. Create Feedback Structure

```
## Strengths

1. [Specific strength with evidence]
2. [Another specific strength]

## Areas for Growth

1. [Area needing improvement]
   - [Specific observation]
   - [Suggestion for improvement]

2. [Another area]

## Next Steps

- [Concrete action to improve]
- [Optional: resource suggestion]

## Overall Rating

[Score/Level if applicable]
```

### 4. Differentiate Feedback Type

**For Essays/Writing**
- Thesis/clarity
- Organization
- Evidence/analysis
- Grammar/mechanics
- Style/voice

**For Math/Problem Sets**
- Problem understanding
- Strategy/approach
- Calculation accuracy
- Explanation/justification
- Efficiency

**For Projects/Performances**
- Planning/preparation
- Execution/quality
- Creativity/originality
- Presentation/communication
- Meeting requirements

### 5. Output and Delivery

- Present feedback in clear, readable format
- Use bullet points for specific comments
- Include quoted student work when referencing
- Keep tone professional yet supportive

### 6. Offer Next Steps

- "Would you like me to generate a rubric for this assignment type?"
- "Shall I create a mini-lesson addressing common errors?"
- "Need a conference notes template for student feedback discussion?"
- "Want to draft a parent communication about this work?"

## Notes

- Focus on actionable feedback, not just correction
- Balance positive and constructive comments (3:1 ratio)
- Use "I notice" language for objectivity
- Prioritize 2-3 key areas per assignment
- Feedback should be useable by the student
